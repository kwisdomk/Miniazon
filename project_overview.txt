Data Structure Specifications
1. Inverted Index - For Search
Purpose: Fast product search and retrieval
Implementation: Hash Table + Sets
Operations:

add_product(product_id, text) - O(k) where k = words

search(query) - O(m) where m = query words

get_suggestions(prefix) - O(1) with prefix tree

Real-world analogy: Amazon product search

2. Shopping Cart - Hash Table
Purpose: User session cart management
Implementation: Dictionary of dictionaries
Operations:

add_item(user_id, product_id) - O(1)

remove_item(user_id, product_id) - O(1)

get_cart(user_id) - O(1)

update_quantity(user_id, product_id, quantity) - O(1)

Real-world analogy: Amazon shopping cart

3. Product Graph - Recommendations
Purpose: "Customers who bought this also bought..."
Implementation: Adjacency List
Operations:

add_relationship(product_a, product_b) - O(1)

get_recommendations(product_id, depth=2) - O(V+E)

find_similar_users(user_id) - O(VÂ²) with matrix

Real-world analogy: Amazon recommendation engine

4. Inventory Queue - Stock Management
Purpose: Handle restocking requests and order processing
Implementation: Queue with priority elements
Operations:

add_restock_request(product_id, quantity) - O(1)

process_next_restock() - O(1)

get_urgent_requests() - O(n) for priority

Real-world analogy: Amazon warehouse management
